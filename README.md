# PostBot — ИИ ведёт твой Telegram-канал

Сайт с личным кабинетом. Пользователь добавляет канал, задаёт тему, стиль, источники и
расписание — ИИ (Claude) сам пишет посты. Их можно проверять и править на сайте, потом
публиковать сразу или по расписанию. Встроен баланс токенов и оплата через YooKassa.

Тебе **не нужно ничего менять в коде**. Только вставить ключи в одном месте (Railway → Variables).

---

## Что где регистрировать (всего 3 сервиса)

| Сервис | Зачем | Ссылка | Платно? |
|--------|-------|--------|---------|
| **Telegram-бот** | его добавляют в каналы, он публикует | [@BotFather](https://t.me/BotFather) | бесплатно |
| **Claude API** | пишет тексты | console.anthropic.com | по токенам |
| **YooKassa** | приём оплаты от клиентов | yookassa.ru | по тарифам YooKassa |

GitHub и Railway у тебя уже есть — на них всё крутится.

## Установка — по шагам

### Шаг 1. Создай Telegram-бота
1. Открой [@BotFather](https://t.me/BotFather), команда `/newbot`.
2. Задай имя и username (username заканчивается на `bot`).
3. Сохрани **токен** (вид: `8123456789:AAH...`) и **username** (без `@`).

### Шаг 2. Получи ключ Claude
1. Зайди на **console.anthropic.com** → **API Keys** → **Create Key**.
2. Скопируй ключ (`sk-ant-...`). Положи на счёт немного денег (Billing).

### Шаг 3. Залей проект на GitHub
1. Создай новый репозиторий на github.com (приватный).
2. Загрузи в него **все файлы из этой папки** (кнопка *Add file → Upload files* — можно перетащить всё разом).

### Шаг 4. Разверни на Railway
1. railway.app → **New Project** → **Deploy from GitHub repo** → выбери свой репозиторий.
2. Добавь базу: **New → Database → Add PostgreSQL**. (Railway сам создаст переменную `DATABASE_URL` — это нужно, иначе данные сотрутся при обновлении.)
3. Открой сервис → вкладка **Variables** → добавь переменные (значения из шагов 1–2):

   ```
   SECRET_KEY            = любая_длинная_случайная_строка_40+_символов
   ANTHROPIC_API_KEY     = sk-ant-...
   TELEGRAM_BOT_TOKEN    = 8123456789:AAH...
   TELEGRAM_BOT_USERNAME = my_postbot
   ```
4. Вкладка **Settings → Networking → Generate Domain** — получишь адрес вида
   `https://postbot-production.up.railway.app`. Это твой сайт.
5. Railway автоматически соберёт и запустит проект. Открой домен — увидишь экран входа.

### Шаг 5. Подключи оплату YooKassa (можно позже)
1. В личном кабинете YooKassa возьми `shopId` и секретный ключ API.
2. Добавь в Railway → Variables:
   ```
   YOOKASSA_SHOP_ID     = твой_shopId
   YOOKASSA_SECRET_KEY  = твой_секретный_ключ
   PUBLIC_URL           = https://твой-домен.up.railway.app
   ```
3. В личном кабинете YooKassa открой **Интеграция → HTTP-уведомления** и добавь endpoint:
   ```
   https://ТВОЙ-ДОМЕН/api/yookassa/notify
   ```
4. Включи событие `payment.succeeded`. Можно также включить `payment.canceled`, чтобы в истории отображались отменённые платежи. URL должен быть на HTTPS; домен Railway подходит.

Если в магазине YooKassa включена обязательная фискализация чеков, добавь также:
```env
YOOKASSA_SEND_RECEIPT=true
YOOKASSA_VAT_CODE=1
```
`YOOKASSA_VAT_CODE=1` означает «без НДС».

Готово. Сайт работает.

## Как этим пользоваться

1. Открой свой домен → **зарегистрируйся** (это твой личный кабинет; новым даётся немного токенов на пробу).
2. **Новый канал** → впиши:
   - **Тему** — про что канал (например: «сделки M&A на рынке РФ — слияния, поглощения, продажи активов»);
   - **Стиль** — как писать;
   - **Источники** (вкладка «Источники») — конкретные сайты/RSS, куда ИИ обязан заглядывать;
   - при желании — ссылку на чужой публичный канал в поле «Писать как существующий канал», чтобы скопировать стиль.
3. Добавь бота **@твой_бот** администратором в свой Telegram-канал (с правом публикации), впиши `@username` канала и нажми **Проверить подключение**.
4. Выбери расписание и режим:
   - **С проверкой** (по умолчанию) — посты ждут тебя во вкладке «Посты», ты правишь и публикуешь;
   - **Без проверки** — публикуются сразу автоматически.
5. Нажми **Сгенерировать пост** для проверки или жди расписание.

## Важно знать

- **Про «отложенные» в Telegram.** Боты технически не могут класть посты в раздел «Отложенные»
  внутри Telegram (это умеет только ручной режим в приложении или вход по номеру телефона).
  Поэтому очередь и редактирование сделаны **на сайте**: пост генерируется → ты правишь его прямо
  в кабинете → жмёшь «Опубликовать сейчас» или «Запланировать на время». Результат тот же, но всё под контролем.
- **Время в расписании — по UTC.** Москва = UTC+3 (для 13:00 МСК ставь 10:00).
- **Токены.** Тратятся на генерацию текста и анализ чужих каналов. Баланс виден сверху справа,
  пополнение — на странице «Баланс».

## Локальный запуск (для проверки на своём компьютере)

```bash
pip install -r requirements.txt
cp .env.example .env     # заполни ключи
python main.py
# открой http://localhost:8000
```

## Идеи для развития в бизнес

- Несколько каналов на аккаунт уже поддерживаются — можно продавать тарифы «N каналов».
- Автогенерация картинок (добавим позже отдельным модулем).
- Реферальная программа: дарить токены за приглашённых.
- Шаблоны ниш (M&A, крипта, недвижимость) с готовыми источниками — ускоряет онбординг.
- Аналитика: какие посты ты чаще публикуешь без правок → дообучать стиль.

## Структура проекта (для разработчика)

```
main.py          FastAPI: API + раздача сайта + запуск планировщика
config.py        все настройки из переменных окружения
database.py      модели и подключение к БД (Postgres/SQLite)
security.py      пароли и токены входа
telegram_api.py  публикация и проверка канала
research.py      сбор источников + чтение чужого канала
generator.py     генерация текста и анализ стиля (Claude)
billing.py       создание и проверка платежей YooKassa
tasks.py         генерация черновиков и публикация
scheduler.py     периодический запуск задач
schemas.py       форматы запросов
static/          сайт (index.html, app.js, styles.css)
```
