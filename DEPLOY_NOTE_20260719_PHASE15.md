# Фаза 1.5 — Яндекс.Поиск для новостных каналов (2026-07-19)

## Что сделано

1. **Новый модуль `yandex_search.py`** — клиент Yandex Search API v2,
   синхронный режим (`POST https://searchapi.api.cloud.yandex.net/v2/web/search`,
   авторизация Api-Key). Парсит base64-XML выдачу: title, url, сниппеты
   (headline + passages), modtime. Ошибки сервиса → `SearchUnavailable`,
   пустая выдача (code=15) → пустой список.

2. **`generator.py`**:
   - `generate_post`: при провайдере yandex и включённом поиске
     (use_web_search без topic, ЛИБО явный topic + use_web_search, и нет
     готовых материалов) — сначала запрос в Яндекс.Поиск, выдача
     подкладывается в промпт как источники («используй только эти факты»).
     Сбой/пустая выдача → тихая деградация в генерацию без поиска
     (прежнее поведение). Стоимость поиска добавляется к токенам поста.
   - `check_news_available`: для yandex теперь реальная проверка — поиск по
     теме канала, свежими считаются документы моложе 3 дней (документы без
     modtime считаются свежими — Яндекс отдаёт дату не всегда). Fail-open:
     сбой поиска не блокирует цикл генерации.

3. **`config.py`** — новые env:
   - `YANDEX_SEARCH_ENABLED` (default true) — рубильник фичи
   - `YANDEX_SEARCH_API_KEY` (default = YANDEX_API_KEY)
   - `YANDEX_SEARCH_MAX_RESULTS` (default 8)
   - `YANDEX_SEARCH_TOKEN_COST` (default 3000) — внутренняя стоимость
     одного поискового запроса в токенах для списания с баланса

4. **`static/legal/privacy.html`** — процессоры обновлены:
   Anthropic → ООО «Яндекс.Облако» (Foundation Models, Search API),
   Railway (серверы ЕС) → ООО «Таймвэб Клауд» (серверы РФ).

5. **`test_yandex_search.py`** — 27 тестов (парсинг, свежесть, интеграция,
   деградация). Все проходят; регрессия test_generator_style.py — зелёная.

(генерация news-каналов без поиска, check_news_available пропускается).
